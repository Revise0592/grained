Grained helps you turn scan deliveries into a searchable archive: import a lab ZIP, enrich each roll with metadata, then browse and annotate frames for years. The workflow stays local and private — self-hosted, no cloud account, no telemetry.

Features

01 — Import

Zip import

Drop a lab scan zip and Grained extracts, sorts, and thumbnails every frame automatically. Supports JPG, TIFF, PNG, HEIC, and WebP.

02 — Metadata

Roll metadata

Log film stock, format (35mm / 120 / 4×5), ISO, push/pull, camera, lens, lab, and develop process — C-41, E-6, B&W, cross-process.

03 — Browse

Lightbox viewer

Full-screen viewer with keyboard navigation, CW/CCW rotation, and a per-frame info panel that keeps your data close at hand.

04 — Annotate

Per-frame data

Frame number, shutter speed, aperture, EV compensation, focal length, and freeform notes — per frame, per roll.

05 — Discuss

Comment threads

Per-roll and per-frame comment threads let you journal your thoughts, remind yourself of settings, or document the shoot.

06 — Manage

Cover, delete & theme

Pin any frame as the roll's catalog cover. Multi-select for bulk deletion. Dark and light theme included — no external accounts, no cloud, no telemetry.

How it works

01

Get your scans

Request a zip of scans from your lab. Most mail-in labs offer this as a standard delivery option.

02

Create a roll

New Roll → drag zip onto the upload zone → name the roll → Import Roll.

03

Enrich metadata

Add film stock, camera, lens, dates, process notes, and per-frame settings in Edit so each roll keeps full context.

04

Browse and archive

Review in the lightbox, leave notes, and keep every roll in your own long-term archive.

Why Grained

Workflow stage What you do What you keep
Import Drop in a lab scan ZIP and let Grained prepare frames automatically. A clean start for each roll without manual file wrangling.
Enrich Capture roll metadata, frame notes, and shoot context as you review. Searchable history with technical details intact.
Browse Use the lightbox and roll views to revisit work quickly. A personal archive that stays easy to navigate over time.
Own it Run it in Docker on your own hardware or server. No hosted account, no third-party cloud dependency, your data stays with you.

Install

Start with Docker,
tune for your deployment.

Grained runs in Docker with persistent local storage. Most setups are quick, but environment values can vary depending on auth, API access, and upload behavior.

Variable Default Description
DATABASE_URL file:/data/grained.db SQLite path inside the container
UPLOAD_DIR /data/uploads Scan and thumbnail storage
AUTH_PASSWORD unset Optional: set for password-protected access; leave unset for open local use.
SESSION_SECRET unset Optional but required with AUTH_PASSWORD; used to sign login sessions.
API_KEY unset Optional: bearer token for API clients and scripted integrations.
UPLOAD_TEMP_TTL_MS 86400000 Optional tuning: how long temporary upload chunks are kept before cleanup (milliseconds).
ENABLE_LEGACY_CHUNK_UPLOAD false Optional compatibility flag for older chunked upload clients.
# 1. Clone
git clone https://github.com/Revise0592/grained.git
cd grained

# 2. Configure
cp .env.example .env

# 3. Start
docker compose up -d

# Open http://localhost:3000

All data — photos and database — lives in the grained_data named volume. Updating Grained is a pull and restart; your archive is never affected.
